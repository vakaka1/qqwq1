# Шаблоны сайтов

Сейчас система ищет шаблоны только в одной папке:

1. Любые `*.html` файлы в папке `site_templates/`.

Что лучше использовать в шаблоне:

- Делать шаблон самодостаточным: один HTML-файл с встроенными CSS/JS.
- Оставлять публичные ссылки только через плейсхолдеры, без прямых IP и внутренних адресов.
- Подставлять значения через плейсхолдеры вида `{{PLACEHOLDER}}`.

Поддерживаемые плейсхолдеры для новых шаблонов:

- `{{SERVICE_NAME}}` или `{{SITE_NAME}}` — название сайта.
- `{{BOT_NAME}}` — имя привязанного бота.
- `{{BOT_USERNAME}}` — username бота с `@`.
- `{{TELEGRAM_URL}}` — ссылка на бота `https://t.me/<username>`.
- `{{CONFIG_URL}}` — URL сайта, который сам выдаёт `site`-конфиг.
- `{{BOT_START_URL}}` — совместимый алиас для `{{CONFIG_URL}}`, чтобы старые шаблоны не ломались.
- `{{PUBLIC_URL}}` — итоговый публичный URL сайта.
- `{{DOMAIN}}` — домен, если он указан.
- `{{TELEGRAM_START_URL}}` — deep-link в привязанный Telegram-бот.
- `{{SITE_TRIAL_DURATION_HOURS}}` — длительность site-теста в часах.
- `{{SITE_TRIAL_TOTAL_GB}}` — лимит трафика site-теста в гигабайтах.

Как добавить новый шаблон:

1. Положить новый файл, например `site_templates/promo-v2.html`.
2. Использовать нужные плейсхолдеры внутри HTML.
3. Открыть раздел `Сайты` в админке и выбрать новый шаблон в шаге выбора шаблона.
